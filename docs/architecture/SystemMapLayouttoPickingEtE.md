Что реально происходило в системе во времени?

End-to-End Flow
Layout draft
  -> validate
  -> publish
  -> generated cells
  -> storage locations available

ImportJob
  -> parse
  -> validate
  -> preview
  -> publish
  -> operational demand / inventory updates available

Storage truth
  + product location roles
  + operational constraints
  -> readiness evaluation

Readiness result
  -> pick task generation
  -> pick execution
  -> confirmation events

All major transitions
  -> domain events
  -> analytics / AI / audit
Critical Boundaries
Must remain separate

spatial truth vs storage truth

storage truth vs operational mapping

readiness vs execution

domain events vs row-level audit

Must remain enforced

published layout is authoritative spatial truth

raw imports do not bypass staging

pick tasks are created from readiness

event history is semantic, not just CRUD history.
