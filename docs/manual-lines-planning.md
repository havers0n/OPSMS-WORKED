# Manual line planning entry points

`/operator/manual/lines` offers three planning entry points:

1. **בנה קווים מביקוש זמין** is the recommended flow. After resolving an active target shift, the web app creates one rolling draft from all available demand and opens Scheme Builder. The target shift ID is authoritative; the URL date is display context only.
2. **מאגר ביקוש** is a visibility and control screen. It does not create or publish a plan.
3. **תכנון ידני מאצווה** is the advanced legacy flow for selecting one imported DataSheet batch before opening Scheme Builder.

Rolling creation and Builder initialization never publish automatically. Publication remains an explicit Scheme Builder action.
